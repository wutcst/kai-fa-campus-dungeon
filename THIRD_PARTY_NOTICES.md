### Planned Reuse Scope

Campus Dungeon may reuse or adapt Pixel Dungeon visual assets and small,
well-scoped implementation ideas to improve the final dungeon presentation.
The preferred reuse scope is:

- dungeon tiles, doors, items, entities, projectiles, and HUD-like visual assets;
- visual style references for room layout, fog, traps, and resources;
- small algorithmic ideas only when they can be adapted into the current
  `cn.edu.whut.sept.dungeon` Java/Swing architecture.

This notice does not import Pixel Dungeon assets or source code by itself.
When actual assets or source files are copied into this repository, the PR must
record the copied paths, original upstream paths, whether the files were
modified, and the reason for reuse.

### Project Boundary

Campus Dungeon must not copy commercial Soul Knight assets or source code. If
Pixel Dungeon source code is copied or modified, the project must be treated as
a GPL-derived work for distribution purposes, and the README and course report
must disclose that scope.
